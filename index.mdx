## Get started

<Steps>
  <Step title="Install a component package">
    Add the global component library and its peer dependencies to your project.

    ```bash
    bun add @canonical/react-ds-global @canonical/styles react react-dom
    ```
  </Step>
  <Step title="Import styles">
    Import the aggregated stylesheet once at your application entry point.

    ```typescript
    import "@canonical/styles";
    ```
  </Step>
  <Step title="Use a component">
    Import and render components using standard named exports.

    ```tsx
    import { Button } from "@canonical/react-ds-global";

    export default function App() {
      return <Button importance="primary" anticipation="constructive">Get started</Button>;
    }
    ```
  </Step>
  <Step title="Explore the full library">
    Browse all available components in the [Global Components](/components/global) reference, or jump straight to [Forms](/components/forms) for form-specific controls.
  </Step>
</Steps>

## Package overview

| Package | Description |
|---------|-------------|
| [`@canonical/react-ds-global`](/reference/react-ds-global) | Universal UI primitives: Button, Card, Badge, Tooltip, and more |
| [`@canonical/react-ds-global-form`](/reference/react-ds-global-form) | Form controls with react-hook-form integration |
| [`@canonical/react-ds-app`](/reference/react-ds-app) | Application-level UI: Navigation, Toolbar |
| [`@canonical/styles`](/reference/styles) | Aggregated CSS stylesheet — import once |
| [`@canonical/ds-types`](/reference/ds-types) | TypeScript modifier family types |
| [`@canonical/utils`](/reference/utils) | Shared utility functions |

<Note>
  All Pragma packages share a single version number. If you use `@canonical/react-ds-global@0.18.0`, every other Pragma package at `0.18.0` is compatible — no compatibility matrix required.
</Note>
