## Prerequisites

Before you start, make sure you have the following installed:

- **Bun 1.3.9 or later** — used for package management and script execution
- **React 19 or later** — required peer dependency for component packages
- **Node.js 22.12 or 24** — required for Storybook and Lerna

<Warning>
  Node 22.x earlier than 22.12 has module resolution issues. Node 23 has a known compatibility issue and should be avoided. Install Node using nvm rather than snap to avoid permission and build issues.
</Warning>

## Installation

<Steps>
  <Step title="Install the component package">
    Add the global component library, styles aggregator, and React peer dependencies to your project.

    ```bash
    bun add @canonical/react-ds-global @canonical/styles react react-dom
    ```

    <Note>
      All Pragma packages share a single version number. You can safely add other Pragma packages at the same version — for example, `@canonical/ds-types` for TypeScript modifier types or `@canonical/react-ds-global-form` for form controls.
    </Note>
  </Step>

  <Step title="Import the stylesheet">
    Import `@canonical/styles` once at your application entry point. This aggregates all CSS layers — design tokens, base element styles, colour modes, and density variants — in the correct order.

    ```typescript
    import "@canonical/styles";
    ```

    Your entry point is typically `main.tsx`, `App.tsx`, or the root layout file depending on your framework.
  </Step>

  <Step title="Render your first component">
    Import components by name from `@canonical/react-ds-global` and use them in your JSX.

    ```tsx
    import { Button } from "@canonical/react-ds-global";

    export default function App() {
      return (
        <Button importance="primary" anticipation="constructive">
          Get started
        </Button>
      );
    }
    ```
  </Step>

  <Step title="Compose a richer example">
    The example below combines `Button`, `Card`, and `Icon` to build a typical UI surface. `Card` uses subcomponents — `Card.Header`, `Card.Image`, and `Card.Content` — composed directly in JSX rather than configured through props.

    ```tsx
    import { Button, Card, Icon } from "@canonical/react-ds-global";

    export default function FeatureCard() {
      return (
        <Card emphasis="highlighted">
          <Card.Image
            src="/images/feature-preview.png"
            alt="Feature preview screenshot"
          />
          <Card.Header>
            <h3>New feature available</h3>
          </Card.Header>
          <Card.Content>
            <p>Deploy to production with a single command.</p>
            <Button
              importance="primary"
              anticipation="constructive"
              icon={<Icon icon="arrow-right" />}
              iconPosition="end"
            >
              Deploy now
            </Button>
          </Card.Content>
        </Card>
      );
    }
    ```

    <Tip>
      The `Icon` component renders SVG icons from `@canonical/ds-assets` using an `<svg><use>` reference. Icons are served as static files — make sure your build tool copies the `icons/` directory from `@canonical/ds-assets/dist` to your public folder, or configure the `rootPath` prop on `Icon` to point to wherever you serve them.
    </Tip>
  </Step>
</Steps>
