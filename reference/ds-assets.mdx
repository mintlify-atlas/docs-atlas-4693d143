`@canonical/ds-assets` contains icons and shared visual assets used across Pragma component packages. All icons use a consistent 16×16 viewBox with `currentColor` fill, so they inherit text colour from their context without any extra configuration.

## Installation

```bash
bun add @canonical/ds-assets
```

## Serving icons

Icons are shipped as individual SVG files in the `icons/` directory of the package. You must expose this directory at a public URL so the `Icon` component can load them at runtime.

```
node_modules/@canonical/ds-assets/icons/
  checkmark.svg
  settings.svg
  close.svg
  ... (150+ icons)
```

With Vite, copy them to your public folder:

```typescript
// vite.config.ts
import { viteStaticCopy } from "vite-plugin-static-copy";

export default {
  plugins: [
    viteStaticCopy({
      targets: [{
        src: "node_modules/@canonical/ds-assets/icons",
        dest: ".",
      }],
    }),
  ],
};
```

Then icons are available at `/icons/<name>.svg`.

## Using icons with the Icon component

Use the `Icon` component from `@canonical/react-ds-global` to render icons. Pass the icon name as a string — TypeScript checks the name against the `IconName` union at compile time.

```tsx
import { Icon } from "@canonical/react-ds-global";

<Icon icon="settings" />
<Icon icon="checkmark" />
<Icon icon="arrow-right" />
```

If your icons are not at the default `/icons` path, set `rootPath`:

```tsx
<Icon icon="settings" rootPath="/static/icons" />
```

## IconName type

```typescript
import type { IconName } from "@canonical/ds-assets";

// Compile-time checked icon name:
const myIcon: IconName = "settings";
```

## Styling icons

Icons use `currentColor` for their fill. Set the icon colour via CSS `color` on the icon or a parent element:

```css
.my-icon {
  color: var(--color-positive);
  width: 1rem;
  height: 1rem;
}
```

```tsx
<span className="my-icon">
  <Icon icon="success" />
</span>
```
