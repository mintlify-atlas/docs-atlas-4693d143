Pragma provides two generator tools for scaffolding new components and packages. Both produce files that follow the established conventions so you start with a consistent structure rather than hand-writing boilerplate.

<Note>
  Generated code has no special relationship to the generator after creation. Rename, restructure, or extend the files freely вЂ” the generator is only involved at scaffold time.
</Note>

## Summon CLI

Summon is a generator framework. The `@canonical/summon-component` and `@canonical/summon-package` packages plug into it to provide component and package scaffolding.

### Installation

<Steps>
  <Step title="Install the summon packages">
    Install the core CLI and the generator packages as global links:

    ```bash
    cd packages/summon && bun link
    cd packages/summon-component && bun link
    cd packages/summon-package && bun link
    ```

    Or add them as dependencies and use `bunx`:

    ```bash
    bun add @canonical/summon @canonical/summon-component @canonical/summon-package
    ```
  </Step>

  <Step title="Verify the installation">
    List all available generators:

    ```bash
    summon --help
    ```

    You should see `component` and `package` listed as available topics.
  </Step>
</Steps>

### Component scaffolding

Run `summon component` from within a package directory to scaffold a new component. The last path segment becomes the component name.

```bash
# React component
summon component react --component-path=src/lib/MyComponent

# Svelte component
summon component svelte --component-path=src/lib/MyComponent

# Interactive mode вЂ” prompts guide you through options
summon component react
```

For a React component at `src/lib/Button`, this creates:

```
src/lib/Button/

в”њв”Ђв”Ђ types.ts             # Props interface extending HTML attributes
в”њв”Ђв”Ђ index.ts             # Barrel export
в”њв”Ђв”Ђ Button.test.tsx      # Unit tests (Vitest + Testing Library)
в”њв”Ђв”Ђ Button.ssr.test.tsx  # Server-side rendering tests
в”њв”Ђв”Ђ Button.stories.tsx   # Storybook stories
в””в”Ђв”Ђ styles.css           # Component styles
```

It also appends to `src/lib/index.ts`:

```typescript

```

#### Flags

| Flag | Description | Default |
|------|-------------|---------|
| `--component-path` | Full path for the component | Interactive prompt |
| `--with-styles` / `--no-with-styles` | Include or skip `styles.css` | `true` |
| `--with-stories` / `--no-with-stories` | Include or skip Storybook stories | `true` |
| `--with-ssr-tests` / `--no-with-ssr-tests` | Include or skip SSR test file | `true` |
| `--dry-run`, `-d` | Preview files without writing them | вЂ” |
| `--yes`, `-y` | Skip confirmation prompts | вЂ” |

### Package scaffolding

Run `summon package` from the monorepo root to scaffold a new package:

```bash
# Interactive вЂ” prompts guide you through options
summon package

# TypeScript tool (no build step, GPL-3.0)
summon package --name=@canonical/my-tool --type=tool-ts

# Publishable library (compiled output, LGPL-3.0)
summon package --name=@canonical/my-lib --type=library

# With React and Storybook
summon package --name=@canonical/my-lib --type=library --with-react --with-storybook

# Preview without writing
summon package --name=@canonical/my-tool --type=tool-ts --dry-run
```

For a `tool-ts` package named `@canonical/my-tool`, this creates:

```
packages/my-tool/
в”њв”Ђв”Ђ package.json      # type: module, main: src/index.ts, license: GPL-3.0
в”њв”Ђв”Ђ tsconfig.json     # extends workspace TypeScript config
в”њв”Ђв”Ђ biome.json        # extends @canonical/biome-config
в”њв”Ђв”Ђ README.md
в””в”Ђв”Ђ src/
    в””в”Ђв”Ђ index.ts      # export entry point
```

#### Package types

| Type | License | Entry point | Use for |
|------|---------|-------------|--------|
| `tool-ts` | GPL-3.0 | `src/index.ts` | CLI tools, scripts, generators, dev utilities |
| `library` | LGPL-3.0 | `dist/esm/index.js` | Shared utilities, component libraries, public packages |
| `css` | LGPL-3.0 | `src/index.css` | Design tokens, CSS utilities, style primitives |

#### Flags

| Flag | Description | Default |
|------|-------------|---------|
| `--name` | Full scoped package name (e.g., `@canonical/my-package`) | Interactive prompt |
| `--type` | Package type: `tool-ts`, `library`, or `css` | Interactive prompt |
| `--with-react` | Add React peer dependencies and JSX config | `false` |
| `--with-storybook` | Add Storybook configuration and scripts | `false` |
| `--with-cli` | Add a CLI binary entry point (`src/cli.ts`) | `false` |
| `--no-run-install` | Skip the install step after creation | вЂ” |

## Yeoman generator

`@canonical/generator-ds` provides a Yeoman-based generator for scaffolding React components.

### Installation

```bash
npm install -g yo @canonical/generator-ds
```

### Usage

Run the generator from within a package directory:

```bash
yo @canonical/ds:component src/MyComponent
```

This creates the same seven-file component structure:

```
src/MyComponent/
в”њв”Ђв”Ђ MyComponent.tsx
в”њв”Ђв”Ђ types.ts
в”њв”Ђв”Ђ index.ts
в”њв”Ђв”Ђ MyComponent.tests.tsx
в”њв”Ђв”Ђ MyComponent.ssr.tests.tsx
в”њв”Ђв”Ђ MyComponent.stories.tsx
в””в”Ђв”Ђ styles.css
```

#### Flags

| Flag | Description |
|------|-------------|
| `--withStyles` | Include `styles.css` (default: on) |
| `--withStories` | Include Storybook stories (default: on) |
| `--withoutSsrTests` | Skip the SSR test file |
